How we test — and how to catch us being wrong

Nothing on this page is secret. There is no proprietary method here, because a checking tool that hides its checking is asking for the same blind trust it warns you about. This page shows every test we run, the bugs we found in ourselves, and what we still can't do. Tap any underlined word for a plain meaning.

The receipts rule (locked): a grade without its arithmetic is an opinion.

Everywhere in this app, when we grade a claim, the number that produced that grade is one tap away. Not sometimes. Not on request. Every time.

The reason is simple. If we print "UNSUPPORTED" and don't show you why, we're just asserting something with a confident face — which is the exact behaviour this whole project exists to warn you about. A grade with no receipt is our opinion wearing a lab coat.

So: if you ever find a grade in this app with no number behind it, that is a bug, and it's the kind we most want reported.

What the grades mean

Our scale never says "true" or "false," and never says anyone is lying. The worst it reaches is "go verify this yourself."

FACT-BACKED
SUPPORTED
UNVERIFIED
STRETCHED
UNSUPPORTED
CONCLUSION

UNVERIFIED is a real position, not a failure. It means: nobody has checked this properly yet, including us. A lot of true things sit here. So do a lot of false ones. The honest move is to say so rather than round it up to "supported" or down to "nonsense."

Why we never say "lying": we can measure what was said and what the numbers show. We cannot see inside someone's head, and a tool that guesses at motive is inventing a fact it doesn't have. We measure the force, not the motive — you decide the intent.

The tests we run on a signal

Every trading rule we check has to get past the same set. They are grouped by the question they answer, not counted as equal votes — because several of them turned out to be asking the same question in different words, and we only found that by testing.

1 · Is there enough data to answer at all? ▾

This runs first, and it can stop everything. We work out the smallest edge this test could reliably have spotted. If that number is bigger than an edge worth acting on, we return "cannot determine" and refuse to grade.

Why refusing matters: "no evidence" and "cannot determine" sound similar and mean opposite things. One says we looked and it isn't there. The other says we couldn't see. A test too weak to see anything will report "nothing here" every time — which quietly confirms whatever the tester already believed. We measured this: 0% of genuinely real edges were ever wrongly reported as "no evidence."

2 · Did it survive randomness? ▾

Four different checks, treated as one question, not four votes:

The blind dart. Pick the same number of random moments and see what pure luck scores. A real signal has to beat luck's usual range.

The shuffle test. Keep the signal's exact firing pattern — every gap, every burst — and slide the whole pattern to a random new point in history. If its timing carries no real information, it should do about as well anywhere.

The overlap correction. A maths fix (Newey-West) for the fact that measurements taken close together aren't independent facts.

The many-tries correction. Test nine rules and one will look brilliant by luck alone. Holm-Bonferroni raises the bar to account for how many we tried.

Why these count as one question, not four: we tested them on markets containing no real pattern at all. Two of them agreed with each other 34 times against 19 disagreements. They are four windows into one room. Calling that "four independent checks agreed" would have told you something four times more reassuring than the truth.
3 · Did it beat the alternatives? ▾

Beat doing nothing. If prices rose 55% of the time anyway, a signal has to beat 55%, not zero.

Beat deliberate nonsense. We run four rules built to mean nothing — "every 17th bar," "the price ends in a 7." A nonsense rule can't know anything, so it should land at the average. Any real signal has to beat what the junk scored.

We attacked this one too, trying to prove one of our "nonsense" rules was secretly picking up something real. The attack failed and the controls held.

4 · Did it hold up under stress? ▾

Across time. We cut the history into chunks and check the signal points the same way in most of them. A rule that only worked in one stretch is describing that stretch, not a pattern.

On separate cases only. Signals fire in bunches, and bunched fires often measure nearly the same stretch of price over and over. We re-run using only cases far enough apart to be genuinely separate stories.

5 · Is it big enough to matter? ▾

A real edge smaller than the cost of trading is not usable. You set the cost yourself; the bar moves with it. A signal can be statistically real and still fail here — and that's not a technicality, it's the difference between a fact and a usable fact.

The tests we run on our own tests

This is the part almost nobody publishes. A test can be broken while looking perfectly healthy, so we check the checkers.

Known-answer markets. We build fake markets where we already know the truth — some with no pattern at all, some with a pattern we planted ourselves. Then we see whether the engine says so. On markets with nothing in them, it should find nothing.

Eight different kinds of market. Our first version was tested on one kind of fake market, and a reviewer pointed out that a result from one simulator describes the simulator, not the engine. Re-measured across eight — including fat-tailed, volatility-clustering, regime-switching and jumpy markets — the same engine ranged from 1.7% to 8.3%. The single number we first reported was wrong to state as a property of the engine.

Rules that must always hold. Checked against many random datasets rather than a few hand-picked ones: a probability must stay between 0 and 1; the same input must always give the same answer; an impossibly good result must produce a small p-value; an impossibly bad one must produce a large one.

Comparison against software we didn't write. The most useful check available to you if you can't read our code. Our maths is run against SciPy — written by thousands of contributors, used in published research worldwide — on identical data. Agreement ranged from 1e-9 to 1e-15 (that is, matching to between nine and fifteen decimal places).

A warning about that last one, from our own near-miss. Our first comparison of the overlap correction passed — while barely exercising the thing it was testing. The corrected and uncorrected answers differed by 1.00×, meaning the test proved almost nothing. Re-run on data that actually stressed it, the result moved from "clearly significant" to "not significant" — and still matched to fifteen decimal places. A passing test on a case that doesn't stress the code is not evidence. If you ever want to check us, that's the question to ask: not "did it pass," but "did the test have any teeth."

What we cannot do

We have no out-of-sample holdout. Every test we run uses history we have already seen. By our own rules that caps us at "measured, not predictive" — we can say what happened, not what will. Anyone claiming otherwise about their own backtest, including us, should be asked this question first.

Passing our tests is not the same as being right about markets. Offline tests show the maths is sound and honestly bounded. Whether a gauge is right about the real world is a separate verdict that only live data can give.

Some published methods we don't yet implement: White's Reality Check, Hansen's SPA test, the Probability of Backtest Overfitting, and automatic lag selection for the overlap correction. Named here rather than left for you to discover.

Our crypto prices are an average across many exchanges. Single exchanges can differ from that average by 1–3% during volatile periods. If you compare our numbers to one exchange's chart and they disagree slightly, that's usually why — not an error in either.

Opinions, if we ever publish them

Everything above is measurement. If this project ever publishes warnings or opinions — a concern the builder holds, an argument about what the numbers might mean — it will follow three rules, set in advance, here, before any such page exists:

1. It will be visually unmistakable. Opinion will never share a page or a style with measurement. You will never have to work out which one you're reading.

2. It gets graded like everything else. An opinion carries a needle position too, and the same receipts rule applies. If a warning is UNVERIFIED, it says UNVERIFIED — with the number that put it there.

3. Waiting for certainty is not the honest option. Our own testing shows this engine can't reliably detect small effects. A concern held to a near-certainty standard would stay unpublished forever, including when it's correct. So a warning may be published as UNVERIFIED with weak evidence shown, clearly labelled as one person's concern. That is more honest than silence — and it's exactly what the UNVERIFIED slot is for.

How to catch us. Three questions that work on us and on anyone else:

1. "Show me the number behind that grade." If it isn't there, the grade is an opinion.

2. "Did your test have teeth?" A test that passes without stressing the thing it tests proves nothing. We nearly shipped one.

3. "What did you try that didn't work?" Anyone showing only their successes is showing you a selection, not a result.
